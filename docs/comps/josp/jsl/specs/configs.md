# JOSP JSL Configs

The JSL instance is initialized and startup using a ```JSL.Settings``` object
(Obj. Oriented Programming) that can contains all properties used by the
JSL instance to determinate his behaviour.
That settings' instance can be load from a file or from a Java Map.

Depending on the JOSP Service implementation those settings can be available or
not to the end user. For example the [JSL Shell](/docs/comps/jsl_services.md)
service use the ```jsl.ylm``` file edited by the end user. On the other hand,
the [JCP JSL Web Bridge](/docs/comps/jcp/core/jslwb/README.md) use a
Java Map with predefined values to initialize all his JSL instances.

Because JSL Library is provided for different languages, more info on
```JSL.Settings``` initialization can be found at the corresponding section to
the JSL implementation pages:

* [Java](../langs/java.md#initialization)
* [HTTP APIs](../langs/http.md#initialization)
* [JavaScript](../langs/javascript.md#initialization)

## Properties list

In both case, the file or the Java Map, can contain following properties.

| Property                                                        | Default value            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|-----------------------------------------------------------------|--------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| jcp.connect<br/>(JCP_CONNECT)                                   | true                     | Set 'false' to prevent JSL instance connection to [John Cloud Platform](/docs/comps/jcp/README.md).                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| jcp.client.refresh<br/>(JCP_REFRESH_TIME)                       | 30                       | Seconds between each JCP connection retry, when JCP is not available.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| jcp.client.ssl<br/>(JCP_SSL)                                    | true                     | Set 'false' to disable SSL on communication with the JCP.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| jcp.client.id<br/>(JCP_CLIENT_ID)                               | ""                       | Valid client id for [JCP authentication](../comm.md#jcp-authentication). Each JOSP Service has his own id shared across all his instances.                                                                                                                                                                                                                                                                                                                                                                                                                               |
| jcp.client.secret<br/>(JCP_CLIENT_SECRET)                       | ""                       | Secret associated to client id for [JCP authentication](../comm.md#jcp-authentication).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| jcp.client.token.authCode<br/>(JCP_CLIENT_TOKEN_AUTH)           | null                     | The auth token for auto-login. The token can be obtained by another JSL instance on user login. It require also the ```jsl.usr.name``` and ```jsl.usr.id``` properties configured.                                                                                                                                                                                                                                                                                                                                                                                       |
| jcp.client.callback<br/>(JCP_CLIENT_CALLBACK)                   | "https://localhost:8080" | The redirect url used by the Auth service to redirect login and logout request to your service. It must match with the callback url configured to given client id on Auth service. For example ```http://localhost:8080``` or ```https://www.mywebsite.com/auth/code```.                                                                                                                                                                                                                                                                                                 |
| jcp.url.apis<br/>(JCP_URL_APIS)                                 | api.johnosproject.org    | Url to use as [John Cloud Platform](/docs/comps/jcp/README.md) entrypoint                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| jcp.url.auth<br/>(JCP_URL_AUTH)                                 | auth.johnosproject.org   | Url to use as [JCP authentication](../comm.md#jcp-authentication) service.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| jsl.srv.name<br/>(JSLSRV_NAME)                                  | ""                       | Human displayable JOSP service name (like "My JOSP service")                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| jsl.srv.id<br/>(JSLSRV_ID)                                      | ""                       | JOSP Service id, if not set it correspond to the ```jcp.client.id```.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| jsl.usr.name<br/>(JSLUSR_NAME)                                  | ""                       | The username for auto-login. It require also the ```jcp.client.token.authCode``` and ```jsl.usr.id``` properties configured.                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| jsl.usr.id<br/>(JSLUSR_ID)                                      | ""                       | The user's id for auto login. It require also the ```jcp.client.token.authCode``` and ```jsl.usr.name``` properties configured.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| jsl.comm.local.enabled<br/>(JSLCOMM_LOCAL_ENABLED)              | true                     | Set 'false' to disable the [JSL Local Discovery System](../comm.md#jsl-local-discovery) and disable the JSL instance [Direct Communication](/docs/features/communication.md#direct-communication).                                                                                                                                                                                                                                                                                                                                                                       |
| jsl.comm.local.discovery<br/>(JSLCOMM_LOCAL_DISCOVERY)          | Auto                     | Discovery system implementation, you can choose between different mDNS/Bonjour implementations. It can be one of the following values:<br/>**Auto** choose the discovery system depending the detected operating system<br/>**Avahi** use the Avahi daemon implementation, common on linux system<br/>**DNSSD** the default MacOS bonjour service<br/>**JmDNS** java mDNS implementation from [jmdns](https://github.com/jmdns/jmdns) library<br/>**JmmDNS* java mDNS implementation from [jmdns](https://github.com/jmdns/jmdns) library (multiple interfaces version). |
| jsl.comm.local.onlyLocalhost<br/>(JSLCOMM_LOCAL_ONLY_LOCALHOST) | false                    | If set to 'true', allows connect only to JOSP objects running on local machine.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| jsl.comm.cloud.enabled<br/>(JSLCOMM_CLOUD_ENABLED)              | true                     | Set 'false' to disable the [JOD Gateway S2O Client](../comm.md#jsl-gateway-s2o-client) and disable the JSL instance [Cloud Communication](/docs/features/communication.md#cloud-communication).                                                                                                                                                                                                                                                                                                                                                                          |

In the [JSL source directory](/src/jospJSL/configs/jsl_default.yml) you can find
some ```jsl.yml``` example file used in the JSL development.
